制作期間約1か月
制作形式チーム制作（3名）
担当役割リードプログラマー
使用エンジンUnity
主要言語C#
使用技術Unity EventSystem, Physics2D, DOTween, Notion, Git

PROJECT OVERVIEW

図形を回転・配置してパズルを解くゲームです。自主制作プロジェクトとして、3名のプログラマーチームで企画から実装まで行いました。 シンプルながら奥深いゲーム性を目指し、直感的な操作感とリアルタイムの視覚的フィードバックを重視して設計しました。

PROJECT MANAGEMENT

このプロジェクトでは、リードプログラマーとして技術面だけでなく、チーム全体のマネジメントも担当しました。

  • タスク管理システムの構築：Notionでタスク管理シートを作成し、全メンバーがリアルタイムで進捗を共有できる環境を整備しました。
  • 適切なタスク配分：各メンバーのスキルと作業量を踏まえ、無理のない担当範囲と期限を設定しました。
  • コード品質の担保：コード規約を統一し、Gitでブランチ戦略とPRレビューを設定することで、最終的な品質を確保しました。
  • チーム連携の促進：定期的な技術相談やタスク調整を通じて、メンバーが気持ちよく開発できる環境づくりを心がけました。

TECHNICAL IMPLEMENTATION

図形生成システム

  • Physics2D.OverlapBoxAllを使用した衝突判定により、既存の図形と重なる位置には新しい図形を生成しない制御を実装
  • ステージ内での生成位置、図形の種類、サイズを管理するシステムを構築

ドラッグ&ドロップシステム

  • Unity EventSystemのインターフェース（IBeginDragHandlerなど）を活用
  • ドラッグ中にリアルタイムで配置可否を判定し、視覚的フィードバックを実装
  • ドラッグ中は描画順序を最前面に変更し、操作感を向上

図形操作・回転ロジック

  • 右クリック長押しによる回転機能を実装
  • 回転後の衝突判定も考慮し、操作の自由度と制約のバランスを調整

シーン遷移演出

  • シングルトンパターン（DontDestroyOnLoad使用）で統一されたFadeパネルを作成
  • DOTweenを使用したUI演出（ホバーやフェード）を実装

CODE MANAGEMENT

  • 全体のコードレビューを実施し、保守性・可読性を向上
  • 冗長なロジックの整理、共通関数化、命名規則の統一を推進
  • チームメンバーとの技術相談を通じて、より良い実装方法を模索

LEARNING & GROWTH

このプロジェクトを通じて、技術面だけでなくマネジメント面でも大きく成長できました。

  • 技術共有の重要性：メンバー間で実装手法を共有し、互いのスキルを高め合う環境を作ることの価値を実感
  • 実践的なチーム開発：タスク管理、進捗共有、コードレビューなど、企業開発で必要なワークフローを経験
  • リーダーシップ：「どうすればメンバーが気持ちよく開発できるか」を常に考え、マネジメント面への興味が深まりました
